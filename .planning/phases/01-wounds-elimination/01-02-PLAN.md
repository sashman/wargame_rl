<objective>
Thread alive-guards through every model iteration site in the environment loop: distance cache, action handler, reward phase manager, VP calculator, opponent policy, renderer, and WargameEnv.step wiring. Dead models must not move, control objectives, or dilute rewards. The env step must compute and pass the all_eliminated flag to terminate episodes on full wipe.

Purpose: Without these guards, dead models continue to participate in gameplay mechanics — moving, capturing objectives, and affecting reward calculations. This plan makes elimination meaningful by excluding dead models from all active play (WOUND-03) and wiring termination into the env step loop (WOUND-05).
Output: All iteration sites guard on is_alive; env step terminates on elimination; integration tests confirm correct behavior.
</objective>

```python
class WargameModel:
    @property
    def is_alive(self) -> bool:
        """True while the model has wounds remaining."""
        return self.stats["current_wounds"] > 0

    def take_damage(self, amount: int) -> None:
        """Reduce current wounds by amount, clamped to 0."""
        self.stats["current_wounds"] = max(0, self.stats["current_wounds"] - amount)

def alive_mask_for(models: list[WargameModel]) -> np.ndarray:
    """Boolean array where True = model is alive."""
    return np.array([m.is_alive for m in models], dtype=bool)
```
